# Лабораторна робота №6: CI/CD Pipeline

## Опис
Цей проєкт є логічним продовженням розробки HTTP API для математичної моделі росту бактерій. У даній роботі було впроваджено автоматизований процес CI/CD (Continuous Integration/Continuous Deployment) для забезпечення якості коду та автоматичної збірки застосунку.

## CI/CD Pipeline
Проєкт автоматизовано за допомогою **GitHub Actions**. Конфігурація пайплайну знаходиться у файлі `.github/workflows/ci.yml`.

### Етапи пайплайну:
1. **Checkout:** клонування репозиторію.
2. **Setup Python:** підготовка середовища (версія 3.11).
3. **Install dependencies:** автоматичне встановлення залежностей з `requirements.txt`.
4. **Run tests:** запуск модульних тестів за допомогою `unittest`.
5. **Build Docker image:** автоматична збірка Docker-образу для подальшого розгортання.



## Технічні характеристики
* **CI/CD інструмент:** GitHub Actions
* **Тестування:** unittest
* **Контейнеризація:** Docker
* **Python версія:** 3.11

## Як працювати з проєктом
Кожна зміна (push) у гілці `main` автоматично запускає пайплайн. Статус збірки можна переглянути у вкладці **Actions** вашого репозиторію.

# Базова функціональність API-сервісу

*Оригінальний опис API сервісу:*
Реалізовано веб-сервіс на базі фреймворку Flask, який надає доступ до математичної моделі росту бактерій через endpoint `/calculate`. Сервіс приймає вхідні параметри у форматі JSON, виконує обчислення та повертає результати у форматі JSON-відповіді.

**Технічні характеристики:**
* **Фреймворк:** Flask
* **Порт:** 5000
* **Метод:** POST
* **Формат:** `application/json`
